# Plan d'Exécution : Lancement du Dashboard "Mon Réseau Local"

C'est parti. Nous allons exécuter le plan validé, étape par étape, en commençant par les fondations de données, puis en activant chaque page du dashboard.

## Phase 1 : Fondations Data (Backend)
Avant de coder les fonctionnalités, il faut que la base de données puisse les stocker.

1.  **Création des Tables Supabase**
    *   `network_opportunities` : Pour stocker les échanges (qui, quoi, points, statut).
    *   `network_matches` : Pour l'historique des rencontres quotidiennes.
    *   `trust_scores` : Pour centraliser la réputation de chaque membre.
    *   `network_availabilities` : Pour les choix de créneaux du lendemain.

2.  **Mise à jour du Schéma Prisma / Types**
    *   Mettre à jour les types TypeScript pour refléter ces nouvelles tables.

## Phase 2 : Le Cœur - Page "Opportunités" (Priorité 1)
C'est la fonctionnalité la plus complexe (la modale) et la plus centrale.

3.  **Connecter la Modale "Ajouter une opportunité"**
    *   Remplacer `MOCK_MEMBERS` par une vraie recherche de membres (`/api/network/members`).
    *   Brancher le bouton "Envoyer" pour créer une vraie ligne dans `network_opportunities`.

4.  **Connecter la Liste des Opportunités**
    *   Afficher les vraies opportunités reçues/données depuis la base.
    *   Activer les boutons "Valider" / "Refuser" (mise à jour du statut en base).

## Phase 3 : Le Quotidien - Page "Accueil" (Priorité 2)

5.  **Connecter le "Match du Jour"**
    *   Créer une logique simple pour simuler un match (ou récupérer le dernier `network_match`).
    *   Activer le bouton "J'ai appelé" (mise à jour du statut du match).

6.  **Connecter "Disponibilités"**
    *   Faire fonctionner la sauvegarde des créneaux en base.

## Phase 4 : La Gamification - Page "Confiance & Score"

7.  **Calcul du Score**
    *   Créer une fonction simple : Score = (Opportunités rendues / reçues) * 5.
    *   Afficher le vrai score calculé sur la page.

8.  **Affichage des Dettes**
    *   Lister les vraies opportunités "validées mais non rendues".

## Phase 5 : Finalisation

9.  **Nettoyage & Tests**
    *   Vérifier que tout s'enchaîne bien.
    *   Supprimer les derniers `MOCK_DATA`.
